# Code Review Rules Reference

## Python 代码审查标准

### 1. 命名规范
- 类名：`CamelCase`
- 函数名：`snake_case`
- 常量：`UPPER_SNAKE_CASE`
- 私有成员：`_leading_underscore`

### 2. 函数设计原则
- 单一职责：每个函数只做一件事
- 短小精悍：最好不超过 30 行
- 参数控制：不超过 3 个参数

### 3. 错误处理
- 优先使用异常而非返回错误码
- 异常应有明确的类型
- finally 用于清理资源

### 4. 安全准则
- 避免使用 `eval()`
- 防止 SQL 注入：使用参数化查询
- 防止 XSS：转义用户输入
- 敏感信息不写入日志

### 5. 性能准则
- 避免在循环中进行 I/O 操作
- 使用生成器处理大数据
- 合理使用缓存
- 避免重复计算
