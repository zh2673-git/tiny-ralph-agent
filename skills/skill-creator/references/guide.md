# Skill Creator Guide

## 如何创建高质量的 Skill

### 1. 好的 Skill 描述示例

**❌ 不好**：
```
description: 代码审查技能
```

**✅ 好**：
```
description: 当用户要求审查代码、检查 bug、分析代码质量或进行代码重构时触发此技能。适用于代码审查、质量分析和改进建议场景。
```

关键点：
- 明确触发条件
- 说明适用场景
- 不要过于宽泛

### 2. 指令编写原则

**清晰性**：
- 使用动词开头（审查、检查、生成）
- 明确输入输出
- 列出执行步骤

**完整性**：
- 覆盖主要场景
- 定义错误处理
- 提供示例

### 3. 脚本 vs 指令

**指令（Instructions）**：
- LLM 遵循的行为规范
- 模糊/创造性的任务
- 需要判断的场景

**脚本（Scripts）**：
- 确定性/重复性任务
- 可以自动化的流程
- 需要精确执行的代码

### 4. 版本管理

```yaml
---
name: my-skill
description: ...
version: 1.0.0  # 遵循 semver
---
```

版本更新规则：
- **patch** (1.0.0 → 1.0.1): bug 修复
- **minor** (1.0.0 → 1.1.0): 新增功能
- **major** (1.0.0 → 2.0.0): 破坏性变更

### 5. 测试 Skill

创建后测试：
1. 加载 Skill：`loader.get_skill("skill-name")`
2. 检查 prompt：`skill.to_prompt()`
3. 验证脚本：`skill.scripts`
4. 实际调用测试
